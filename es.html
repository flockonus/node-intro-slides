What?

"Node.js is an open source, cross-platform runtime environment for server-side and networking applications based on Javascript language"

It's currently popular for creating web servers
API, website, webapp, real-time, &+

When?

  • Born in 2009 as a hack on top of engine from google chrome (V8)
  • Sponsored and trademarked by Joyent.
  • solid low level APIs
  • outstanding package manager
  • inclusive community

Nowadays

Considerable part of the Node.js community forked the main project from Joyent, and is now publishing a parallel branch with the name of IO.js

Why using?

Recommended when you are:

  • ok/like Javascript
  • prototyping ideas
  • iterate fast
  • real time apps
  • mobile/web API
  • isomorphic websites

Why not?

(No Hard Rules) not Recommended for:

  • you really dislike Javascript
  • when things cannot break (building a bank/rocket/satellite)
  • when you can afford to focus only backend
  • dependency on libraries exclusive for other language

How does it look like?

Example echo server


var express = require('express')
var app = express()

app.get('*', echo);

function echo(req,res){
  res.send({
    path: req.path,
    query: req.query
  })
}

app.listen(3000);
console.log('Express view app listening port 3000');
